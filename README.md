ğŸ“Œ Architecture Summary
Below is a structured explanation of the provided architecture diagram.

### 1. User Input Layer
The system begins with a simple user interaction:

Upload a PCAP file or a log file
Trigger auto-analysis

The root agent orchestrates the entire pipeline and delegates tasks across specialized agents.

2. Data Processing Pipeline
Once a file is uploaded, the root agent routes it into the Data Processing Pipeline, which includes:

PCAP Parser Agent

Extracts packets
Identifies protocols, endpoints, ports
Reconstructs flows and sessions
Prepares structured features for downstream analysis

Log Parser Agent

Parses system logs, security event logs, access logs, and application logs
Normalizes timestamps, IPs, events, and metadata
Extracts anomaly-relevant attributes
This stage converts raw unstructured or binary data into structured machine-readable formats.

3. Analysis Pipeline
After data processing, three parallel agents perform deeper analysis:

Attack Detection Agent

Uses rule-based, anomaly-based, or ML-based methods
Detects known attack signatures
Evaluates suspicious patterns
Flags high-risk interactions (e.g., port scans, brute force, data exfiltration attempts)

Vulnerability Agent

Identifies vulnerable services
Highlights insecure configurations
Maps findings to CVE/CWE data
Detects outdated protocols or weak cryptography
Timeline Agent
Reconstructs event sequences
Maps chronological attack/incident progression
Helps analysts understand â€œwhat happened and whenâ€
These three agents collectively build a comprehensive forensic view of network or system events.

4. Intelligence Enrichment Pipeline

Insights generated by the analysis agents flow into:
Threat Intelligence Agent
Enriches IPs, domains, hashes, or artifacts
Queries external or internal threat intelligence sources
Maps malicious indicators (IoCs) to known threat actors or campaigns
This enrichment stage transforms raw forensic findings into actionable intelligenceâ€”critical for enterprise SOC workflows.

5. Dashboard Pipeline

Finally, analysis results pass into:
Dashboard Generator Agent
Converts structured results into a live visual dashboard
Generates charts, tables, timelines, and summaries
Creates a shareable Live Dashboard URL
The dashboard serves as the final output for:
Incident reports
Executive summaries
SOC monitoring
Classroom demonstrations
Forensic documentation
